Can small local models drive these databases?

We handed free, laptop-scale language models the same instructions any assistant gets, asked them research questions, and executed whatever SQL they wrote against the live databases through the public endpoint. No cloud AI involved. Here is what happened.

01 Method

Blind questions, real execution, exact-match scoring

Three open-weight models — run locally through Ollama, temperature 0 — each received a condensed version of the project's skill instructions (schema, value vocabularies, analysis rules) and eight research questions, one at a time. Each model's SQL was executed verbatim against the live databases through the same public read-only endpoint available to everyone. An answer scored a strict pass only if it reproduced the value from a pre-registered reference query; answers that differed because the model applied the documented corpus rules more strictly than the reference were separately classified as defensible.

Questions covered corpus counts, joins, value-vocabulary filters, ranking, and use of the built-in search functions, across both databases. Full harness, questions, and raw results are in the repository (migration/08_eval_local_models.py, eval_results.json).

The most instructive finding: capability did not scale with size on this task. The 2.1 GB model answered in under a second per question and matched the 20 B model. The catalog, value vocabularies, and analysis rules in the skill files appear to do the heavy lifting — when the instructions are precise, a small model following them is enough.

03 A worked example

What "defensibly stricter" looks like

Question 4

"How many empirical articles in the SWRD used qualitative methods?"

Reference answer: 15,164 (all years). Model answer: 14,921.

The model's SQL (gemma4:e4b) 
SELECT count(id) FROM swrd.papers
WHERE is_empirical = TRUE
  AND research_method = 'Qualitative'
  AND publication_year >= 1989
  AND is_scientific = TRUE
  AND abstract IS NOT NULL

The model added the corpus-scoping filters the skill file instructs analysts to use by default — a more careful answer than the reference query, and the reason it "failed" strict scoring.

04 Honest limits

What this evaluation does and doesn't show

It shows that free, local, laptop-scale models can reliably translate research questions into correct queries against these databases when given the skill files — the design goal of this project's documentation.

It doesn't show performance on long multi-step analyses, semantic-search orchestration, or robustness across phrasings — eight questions, one run, temperature 0. Treat it as an existence proof, not a benchmark.

Reproduce it: python migration/08_eval_local_models.py <ollama-model> … — the harness, questions, reference queries, and raw outputs are all in the repository.
